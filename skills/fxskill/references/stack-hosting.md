# 域名、服务器与上线

## 先判断

1. 目标用户在哪个地区，对访问速度有什么要求。
2. 中国大陆或海外服务器/网站发布路线是否符合市场、备案和数据要求。
3. 网站是静态/前端、全栈、CMS、电商还是媒体站。
4. 是否需要数据库、后台任务、长连接、事务邮件或对象存储。
5. 是否偏好网站发布平台，还是愿意承担服务器运维。
6. 完整访问链路是否使用中国大陆境内服务器、CDN、边缘节点或其他接入资源，并涉及实名、ICP备案及后续要求。
7. 启动预算、每月预算、预期流量和可接受的故障程度。

先使用已有上下文。缺失信息会改变主要路线时，只追问其中影响最大的一个；其余使用明确假设或条件分支继续。最多一个问题。可以合理假设时，标明假设后继续。

## 域名、DNS 与邮箱

- 域名应便于拼写、口述和记忆，并检查商标、同名品牌和主要社交账号冲突。
- 注册商与 DNS 服务商可以不同；选择前核验续费价、转移规则、DNSSEC、2FA 和账号恢复流程。
- 企业邮箱不应依赖临时转发；按团队规模、投递率、管理能力和用户地区选择正式邮箱服务。
- 域名、云平台、支付和邮箱使用不同的强密码，开启 2FA，并离线保存恢复码。
- 品牌保护应与冒用风险成比例，不默认批量注册所有后缀。

## 服务器与网站发布路线

用户口语里说“托管”时，统一按“服务器”或“不需要自己维护服务器的网站发布平台”理解。

当用户只问网站做好后放在哪里，或表示不懂“托管”，且没有提供网站功能时，先用一句短纯描述解释用户引用的“托管”与“服务器”的关系，例如“你说的‘托管’，就是把网站放到服务器上运行。”；再把网站功能作为唯一决定服务器路线的缺口，输出一个问题。网站功能尚未提供时，这两句是全部回答；问题可直接询问网站是纯展示页，还是需要登录、数据库或支付等功能。在用户回答前，不得给出其他事实、平台或路线，不得给行动，不得提及 ICP 或其他政策事实、来源或日期，也不得使用“现状”“下一步”模板。

| 路线 | 常见选择与适用情况 | 推荐前必须核验 |
| --- | --- | --- |
| 静态/前端 | Cloudflare Pages、Vercel、Netlify；适合静态站、前端应用和轻量函数 | 构建限制、带宽、函数、可用区域、正式套餐与续费规则 |
| 全栈/轻量 SaaS | Vercel、Render、Railway 等；适合需要 API、数据库或少量任务的应用 | 数据库、后台任务、日志、休眠/缩容、备份和迁移方式 |
| 内容站 | CMS 服务或具备可靠备份的主机；适合持续编辑和发布 | 更新、安全、缓存、编辑权限、备份及恢复演练 |
| 中国大陆用户 | 以真实网络测试比较中国大陆云与海外网站发布平台或海外服务器 | 完整链路中的境内接入资源、实名、ICP备案、实际速度、付款方式和运维负担 |
| 全球用户 | 数据附近的区域加 CDN；适合跨地区访问 | 数据驻留、跨区延迟、日志保留、出站流量和跨区成本 |

极短路径：若仅静态且无后端，就比较不需要自己维护服务器的网站发布平台；若需数据库、任务或长连接，就比较全栈网站发布平台或服务器方案；若链路含境内接入资源，就先核验备案，未包含时也由候选接入商确认，不能只看源站位置。

不要只按首月价格推荐。总成本至少包括续费、计算、数据库、存储、流量、备份、日志、邮件和支持。迁移出口必须核验数据完整导出、独立备份、恢复演练、出站费用和停服后的导出窗口。

## 输出方式

直接服务器专题固定使用“现状”和“下一步”两个标题。“现状”只写用户已提供的当前功能需求、预算或运维能力及其直接影响，不出现平台名、候选、价格、来源、候选特有额度、未来路线或“自建服务器”等未选替代路线，也不用“预算足够”“主要限制来自”等词提前给路线结论。“下一步”放一个主要路线；只有主路线存在明确不适用条件时，才增加一个备选。主路线和备选各占一个原子行动；平台名、来源、适用条件、每月成本组成、最大限制和迁移难度全部放在对应行动项的描述中。描述使用当前状态或名词性标签，不得把“保留源码”“配置备份”等额外操作藏成迁移条件。不适用条件必须与所引当前官方来源的产品范围和限制直接匹配；不得把“静态资源请求免费且不限量”扩大为“带宽不限量”，也不得把某项产品限制泛化成整个平台缺少数据库等能力。迁移难度若是基于当前架构的推断，必须明确标注为推断，不冒充平台官方结论。用户只问域名时，只回答域名、DNS、邮箱及与其直接相关的备案条件，不展开服务器百科。

发送前逐字检查“现状”：出现平台名、价格、来源、候选额度、未选路线，或“无需”“不需要”“没有必要”“适合”“不适合”“预算足够”“主要限制来自”时必须重写。纯静态需求可写“当前需求仅包含静态文件分发，未包含数据库或后台任务”，不能写“无需数据库或服务器”。

具体推荐遵循 [source-policy.md](source-policy.md)，并在推荐当日重新核验官方定价、限制和区域。官方核验入口（2026-07-13）：[Cloudflare Pages](https://developers.cloudflare.com/pages/)、[Vercel](https://vercel.com/docs/pricing)、[Netlify](https://docs.netlify.com/manage/accounts-and-billing/billing/)、[Render](https://render.com/docs/service-types)、[Railway](https://docs.railway.com/)。
