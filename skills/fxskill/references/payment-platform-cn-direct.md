# 中国大陆官方直连支付档案

最后批量核验日期：2026-07-15。

同一个支付机构会同时提供多个支付产品，浏览器场景、入口、主体资格和审核材料并不相同。回答时不要把“支持微信支付”“支持支付宝”“支持银联”当成统一能力；必须落到 Native 扫码、H5 手机浏览器、JSAPI 微信内网页、电脑网站支付、AI 订阅、AI 按量、网银网关或云闪付远程支付等具体产品。

## 微信支付

- 当前状态：正常运营
- 平台类型：官方直连
- 运营主体：财付通支付科技有限公司（微信支付/财付通体系）。
- 支付角色：支付机构与商户支付平台；普通商户模式下商户是收款方，微信支付提供下单、收银、清算、账单、退款和通知能力。
- 资金路径：买家微信零钱或绑定银行卡 -> 微信支付商户号 -> 财付通支付清算体系 -> 商户号关联结算银行卡
- 卖家地区：中国大陆商户；具体准入随产品和主体类型变化。
- 支持主体：Native、JSAPI、小程序、付款码等基础权限可覆盖企业、个体工商户、事业单位、政府机关、社会组织等主体，且小微商户可申请 JSAPI、小程序、Native 和付款码；H5 支付公开准入只列企业、事业单位、政府机关、社会组织，个体工商户和小微商户不在 H5 支付支持范围。不能笼统写“个体户支持微信支付”，必须按 Native/H5/JSAPI 等场景拆开判断。
- 买家地区与币种：中国大陆微信支付用户，人民币；本档案不覆盖跨境微信支付。
- 支付方式：Native 原生支付用于桌面网站或其他页面展示微信二维码；JSAPI 网页支付用于微信客户端内打开的商户 HTML5 页面；H5 支付用于手机浏览器网页且不支持在 App 内使用；小程序支付用于商户小程序；App 支付用于商户 App，不作为普通网站默认路线。
- 适合商品：实体商品、普通本地生活服务、SaaS/软件开发/其他互联网服务、课程培训等可按对应行业入驻的业务。
- 限制商品：违法违规业务、金融投资、游戏、在线音视频、在线图书/音乐、网络直播、虚拟充值、彩票、医疗药品等需要按官方行业属性和特殊资质表补充资质；AI 图像额度、虚拟权益、数字内容包应先核对是否归入“软件开发/其他互联网服务”或虚拟业务类目。
- 单次支付：支持。Native 下单 API 生成 code_url 二维码；H5、JSAPI、小程序、App 有各自下单和调起链路；支付结果必须结合异步通知、查单和 T+1 账单核对。
- 订阅与自动续费：不是基础网站支付默认能力；需要按微信支付免密服务/自动续费规则申请并获得用户授权，不得默认勾选、提前扣款、超额扣款或把单次支付入口隐藏。
- API、SDK、插件和支付链接：APIv3 需要商户号、APPID 绑定、商户 API 证书、签名验签、回调解密；Native、H5、JSAPI 等均有独立接口文档。官方直连网站支付以 API 下单生成二维码链接、H5 跳转链接或 JSAPI 调起参数为主，本档案不把它认定为免开发支付链接。
- 平台服务费：无此项；本档案按官方直连支付机构记录，不另列第三方平台服务费。
- 支付通道费：未公开统一数字（以后台显示为准）
- 固定费用、提现费、退款费和拒付费：固定费用：未公开统一数字（以后台显示为准）
  提现费：0
  退款费：未公开统一数字（以后台显示为准）
  拒付费：未公开统一数字（以合同为准）
- 结算：新资金流交易当天实时结算至商户平台基本账户；商户可将资金提现至商户号关联银行卡。
- 结算周期与最低结算额：官方客服页列示新资金流 T+1 到商户平台基本账户；自动提现可按日、周、月设置，资金预计在发起提现后约 2 小时内到账，手动提现次日起 1-3 个工作日留意到账；留存额、最低转出额和行业差异以商户后台显示为准。
- 申请材料：商户主体证照、法定代表人/经营者资料、结算银行卡、已认证 APPID 载体；H5 支付还需支付域名、ICP备案截图、经营场所简介、首页和经营场景截图、主体关系证明、特殊行业资质等。
- 接入路径：先申请微信支付商户号和产品权限；桌面网站优先核对 Native，微信内网页核对 JSAPI，手机浏览器核对 H5；H5 从商户平台“产品中心 -> H5 支付 -> 申请开通”提交域名和 ICP 材料，Native 从商户平台开通 Native 权限并绑定 APPID，随后配置 APIv3 证书、回调、查单和账单下载。
- 退款和售后：Native API 列表包含退款申请、退款查询、退款结果回调；免密服务规则要求商户履行售后和投诉处理义务。
- 数据导出与迁移：微信支付提供交易账单、资金账单和下载账单接口；支付回调和查单指引要求 T+1 获取 T 日交易账单并与商户系统订单核对。
- 实际优点：官方已证实：产品文档覆盖 Native/H5/JSAPI、查单、回调、账单、退款和签名验签，适合有开发能力和合规主体的中国大陆网站作为候选直连能力继续核验。实践证据不足，不能据此判断审核体验或真实到账稳定性。
- 实际问题：官方已证实：H5 有主体限制、支付域名 ICP 与主体关系、特殊行业资质、APPID 绑定和自动续费规则。尚未确认：AI/虚拟数字内容在 Native、JSAPI、H5 下的准确类目、核准费率和结算限制。
- 推荐等级：仅作线索
- 适用条件：作为官方直连线索适用于中国大陆企业或符合具体产品权限的个体工商户，面向中国大陆买家收人民币单次款，能按真实浏览器场景选择 Native、JSAPI 或 H5，并能准备 ICP、APPID、类目资质和 API 接入；只有产品权限、类目、费率和结算已由商户后台、合同或官方书面结果确认后，才能作为正式接入路线。
- 不适用条件：自然人直接收款；个体工商户或小微商户要接 H5 手机浏览器支付；没有 ICP 或页面不能公开展示经营内容；AI 图像额度、虚拟权益、课程、音视频、游戏等类目无法提供对应资质时。

## 支付宝

- 当前状态：正常运营
- 平台类型：官方直连
- 运营主体：支付宝（杭州）数字服务技术有限公司（支付宝开放平台官网页脚主体）；AI 支付开放平台、AI 网页应用收款、AI 订阅 SaaS、AI 按量付费的正式收单或签约主体尚未在公开产品页、文档或申请页中直接写明，仍以签约页或合同载明主体为准。
- 支付角色：支付机构与开放平台；商家是收款方，支付宝提供收银台、统一收单、退款、查单、账单下载、AI 订阅和 AI 按量等支付能力。
- 资金路径：买家支付宝余额、银行卡或花呗等支付工具 -> 支付宝统一收单或 AI 支付收银台 -> 支付宝清算体系 -> 卖家签约支付宝商家账户或合同约定结算账户
- 卖家地区：中国大陆商家；本档案只覆盖面向国内支付宝用户的人民币收款。
- 支持主体：官方已证实：AI 网页应用收款明确支持个人收款场景和个人开发者；该结论只限 AI 网页应用收款，不能扩大为所有支付宝网站支付、AI 订阅 SaaS 或 AI 按量付费都支持个人。AI 订阅 SaaS 接入文档的准入条件仍列示支付宝企业账号。具体自然人账号、AI/数字商品类目、费率和结算仍以签约页、后台、合同或官方书面结果为准。
- 买家地区与币种：中国大陆支付宝用户，人民币；可使用支付宝余额、银行卡、花呗等支付宝收银台支持的支付工具。
- 支付方式：AI 网页应用收款覆盖电脑网页和手机浏览器网页，电脑网页支持账号登录和扫码，手机网页可衔接支付宝客户端或网页收银台；AI 订阅 SaaS 支持周期订阅、席位、按量自动续购；AI 按量付费使用 402 Payment Required/x402 链路；开放平台网页/移动应用仍提供网页和移动端安全支付能力。
- 适合商品：平台官网声称覆盖 AI SaaS 会员、AI 创作工具订阅、视频/图片生成积分与订阅混合计费、API/Skill/MCP 按调用计费、数字内容、课程、普通电商和本地生活网页订单；能否正式用于具体主体和类目仍以签约页、后台、合同或官方书面结果为准。
- 限制商品：AI、数字内容、课程、虚拟商品、游戏、音视频等应按支付宝签约类目和资质审核结果处理；传统电脑网站旧文档要求网站 ICP 备案、页面可访问、经营内容和商品信息完整，并提示个体工商户部分行业受限。
- 单次支付：支持。AI 网页应用收款接入指南使用 alipay.trade.page.pay 发起电脑网站页面支付，并要求以异步通知或查询接口结果为准。
- 订阅与自动续费：平台官网声称支持。AI 订阅 SaaS 官方文档列出个人与团体订阅、周期扣款、席位增减、低代码收银台、订阅升降级、续费失败策略和按量自动续购；接入文档准入条件明确指向支付宝企业账号和联系销售提交信息，个人开发者或个体工商户是否能正式签约尚未确认。
- API、SDK、插件和支付链接：支持官方 SDK、alipay.trade.page.pay、alipay.trade.query、alipay.trade.refund、alipay.data.dataservice.bill.downloadurl.query；AI 网页、AI 移动、AI 按量均提供 `npx -y @alipay/alipay-aipay@latest install` 的 Skill 接入路径；AI 订阅低代码方案可由后台配置价格表并通过接口返回 checkout_url。
- 平台服务费：无此项；本档案按官方直连支付机构记录。AI 订阅或按量产品若另有商业报价，应以报价单或合同记录，不把营销页示例写成统一平台费。
- 支付通道费：未公开统一数字（以报价单为准）
- 固定费用、提现费、退款费和拒付费：固定费用：未公开统一数字（以报价单为准）
  提现费：未公开统一数字（以后台显示为准）
  退款费：AI 网页应用收款未发现单独退款操作费公开数字；退款时已扣交易手续费不退回（官方 AI 网页应用收款接入指南，2026-07-15 核验；仅限 AI 网页应用收款口径，不外推到支付宝其它产品）
  拒付费：未公开统一数字（以合同为准）
- 结算：AI 产品页面只公开“交易、退款和结算体验”等能力，未给出统一公开结算周期；传统电脑网站旧文档写交易款项到商户支付宝账户。正式建议应以签约页、后台或合同显示的结算账户和周期为准。
- 结算周期与最低结算额：AI 网页应用收款、AI 订阅 SaaS、AI 按量付费的公开页没有统一结算周期和最低结算额；具体周期、最低额、提现规则和发票能力以签约页、后台显示或业务人员报价单为准。
- 申请材料：AI 网页应用收款可从一站式接入提交当前账号、商品类目和网站资料；AI 订阅 SaaS 文档要求支付宝企业账号并通过链接提交信息等待解决方案专家联系；网页/移动应用开放平台路径是创建应用、开发配置、提交审核和上线；传统电脑网站旧文档要求营业执照、主体一致的支付宝账户、ICP备案和完整网站商品信息。
- 接入路径：普通网站单次收款可先用 AI 网页应用收款或开放平台网页应用作为官方核验入口；自然人使用 AI 网页应用收款时，从一站式接入提交当前账号、商品类目和网站资料，以后台申请结果确认能否开通、费用和结算。AI 订阅 SaaS 联系销售；AI 按量付费按接入指南返回 402 账单、验支付凭证并发送履约回执；其他主体和产品仍以各自签约页、后台、报价单或合同为准。
- 退款和售后：AI 网页应用接入指南列出 alipay.trade.refund 和退款查询；交易发生后 12 个月内可发起退款，资金原路返回用户账号，退款时手续费不退回，退款信息以接口或退款查询为准。
- 数据导出与迁移：AI 网页应用接入指南列出 alipay.data.dataservice.bill.downloadurl.query，可获取商家离线账单下载地址；迁移到其他通道时需保留本地订单、订阅状态、价格表和履约记录。
- 实际优点：官方已证实：2026-07-14 更新的 AI 支付文档覆盖网页收款、订阅 SaaS、按量付费和 Agent 支付，并给出 alipay.trade.page.pay、异步通知、查单、退款和账单等接口链路。平台官网声称这些产品适合 AI 原生业务先从产品形态选型。实践证据不足，不能据此判断真实签约通过率、审核体验或到账稳定性。

## 银联

- 当前状态：限定开放
- 平台类型：官方直连
- 运营主体：中国银联股份有限公司。
- 支付角色：银行卡组织、清算网络和全渠道支付处理方；商户服务网站手册说明成员机构为收单主体，银联提供收单业务管理和交易处理，不把银联写成小商户的直接收单方。
- 资金路径：买家银行卡、云闪付或网银账户 -> 收单机构商户号 -> 中国银联全渠道支付与清算网络 -> 收单机构合同约定的商户结算账户
- 卖家地区：中国大陆商户；现实路径通常先由收单机构入网、签约和审核，再接入银联开放平台或全渠道接口。
- 支持主体：网银支付业务直联商户接入指引适用于个人网银支付业务和企业网银支付业务的直联商户及其收单机构；云闪付远程支付指引写明商户联系收单机构提交入网申请，收单机构负责真实性和合规性审核。公开资料没有给出自然人或个体工商户自助直连入网结论，小商户现实路径应优先找银行/收单机构/服务商。
- 买家地区与币种：中国大陆银行卡、云闪付或网银用户，人民币；本档案不覆盖境外发卡、跨境清算或外币。
