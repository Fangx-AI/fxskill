# 设计、页面与转化

## 诊断顺序

先使用已有上下文。只有信息缺口会改变判断时，才补问一个影响最大的决策问题；若仍能提供帮助，声明暂定假设后继续，不连续抛出四个问题。

1. 对落地页、转化页或单任务页，唯一目标是什么；对仪表盘、文档站、交易市场或复杂工具，根据页面类型确定一个主要任务和必要的次要任务。
2. 访客必须确认哪些信息，才会继续行动。
3. 当前主要问题属于信息层级、信任、流程，还是视觉一致性。
4. 是否已有真实文案、界面截图、案例、价格和品牌资产。

先修结构和内容，再处理颜色与装饰。不要根据模糊的“高级感”或“科技感”直接推荐风格。

## 页面结构

| 区域 | 要求 |
| --- | --- |
| 落地、转化或单任务页首屏 | 说明是什么、为谁服务、解决什么问题，并提供一个主要行动 |
| 功能 / 服务 | 按用户任务组织，而不是按内部部门或技术名词罗列 |
| 信任信息 | 使用真实界面、案例、数据、团队信息和明确保障 |
| 价格 | 解释计价单位、包含内容、限制、退款和试用条件 |
| FAQ | 回答真实异议，不堆砌关键词 |
| 页脚 | 提供联系方式、条款、隐私政策和必要的主体信息 |

单任务工具的输入与输出应靠近首屏，不能用营销图片遮挡核心工具。仪表盘、文档站、交易市场和复杂工具应按页面类型保留必要的次要任务。

## 视觉规则

- 先确定字体层级、间距、版心宽度和组件状态，再选择颜色。
- 主色用于主要行动，状态使用语义色，避免单一色系主导整个界面。
- 保证正文可读性、对比度、触控目标和键盘焦点清晰。
- 移动端应重新安排优先级，而不是缩小桌面版。
- 没有用户截图、页面测量或现有设计令牌时，不给任何精确尺寸、间距或圆角数值，也不能仅靠“经验起点”继续给数字。

## 资源选择

默认最多推荐一个与当前问题直接相关的参考方向：

- 灵感：Land-book、Lapa Ninja、Awwwards。先核验可用性，只借鉴方向，不复制成品。
- 产品流程：Mobbin、Page Flows，用于研究登录、支付和空状态等流程。
- 字体：Google Fonts、Fontsource；检查中文覆盖、目标地区可用性、许可、隐私、第三方字体服务或自行部署字体文件、页面重量和更新负担。
- 配色：Coolors、Adobe Color；选色后仍需测试对比度。
- 图标：Lucide、Iconify；保持风格一致并核验许可。
- 图片：Unsplash、Pexels；版权许可不等于人物或模特授权。商业页面还要确认财产与艺术作品权利、商标使用和是否构成暗示背书，不用无关氛围图代替产品本身。

具体资源的事实与推荐遵循 [source-policy.md](source-policy.md)。

## 输出方式

直接设计专题的默认正文最多 350 个中文字符。使用“最大问题”作为纯描述诊断区、“核心修改”作为唯一行动区；先指出当前最大问题及其影响，再给出最多三条单句核心修改。每条只使用一个主要谓语动作，不用“并、再、同时、以及、；”连接额外操作。列表后不得补充按钮、图标、报错、状态或其他操作要求；坑只描述风险，不构成额外命令。最多给一个直接相关的参考方向。除非用户明确要求，不输出完整品牌手册。
