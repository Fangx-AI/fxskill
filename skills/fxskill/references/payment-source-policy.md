# 支付资料来源规则

## 证据等级

### A 级：官方直接证据

- 当前价格页、产品规则、开发文档、服务条款、支持地区、申请入口、资质说明。
- 维护者提供的后台截图、申请结果、报价单、合同或官方书面回复，必须附日期。
- A 级证据可以支持费用、资质、地区、能力、结算、产品状态、支付角色和资金路径。

### B 级：官方间接证据

- 官网首页、帮助中心导航、营销页、案例页、面对潜在客户的公开说明。
- B 级证据可以证明平台声称提供某项服务，不能单独证明具体主体、类目、费率、资金路径或结算规则一定适用。

### C 级：多方实操证据

- 至少两个相互独立、时间较新的接入记录、订单截图、后台截图、开发者文章或讨论。
- C 级证据只用于补充审核体验、到账速度、迁移、退款、拒付和客服处理等官方文档难以直接说明的事实。

### D 级：单点经验或不可复核说法

- 单个帖子、转述、无时间的经验贴、联盟内容、销售口头承诺。
- D 级只能作为线索进入待核验清单，不能直接形成正向推荐。

## 易变字段

以下字段在给出正式建议前必须按回答当天重新核验，并在档案中写明 `最后核验日期`：

- 支付角色、卖家地区、支持主体、买家地区与币种、支付方式。
- 平台服务费、支付通道费、固定费用、提现费、退款费、拒付费。
- 结算、结算周期与最低结算额、当前状态、申请入口、产品可用性。

## 直接来源规则

- 费用、资质、地区、产品能力、结算规则、运营状态、支付角色和资金路径，默认必须有 A 级直接来源。
- 没有直接来源时，不把推断写成事实；用结构化待核验块保留不确定性。
- 同一机构下的相邻产品页不能自动替代目标产品页。结论必须和具体产品、主体、地区完全对应。

## 资金路径规则

- `资金路径` 必须严格写成四段：`买家 -> 收款方 -> 清算方 -> 卖家具体账户`。
- 四个节点都必须非空，且不得用 `未知`、`待确认`、`不明` 等占位词伪装成节点。
- 找不到清算方直接证据时，应精确指出缺失的清算节点，例如 `协议约定的结算处理（公开资料没有写明具体清算方）`；这仍然属于未知，只能配 `仅作线索` 或 `已停止/归档`，并保留结构化 `待核验问题`。
- `支付角色` 进入关键字段集合，必须明确说明平台在链路中的角色，不能写成模糊值。

## 推荐联动规则

- 关键字段包括：当前状态、支付角色、资金路径、卖家地区、支持主体、买家地区与币种、支付方式、平台服务费、支付通道费、固定费用、提现费、退款费、拒付费、结算、结算周期与最低结算额、适用条件、不适用条件。
- 任何推荐等级只要关键字段存在模糊未知，就必须填写结构化 `待核验问题`。
- `待核验问题：无` 只允许出现在关键字段全部已知时；否则视为契约错误。
- `直接推荐` 额外要求没有任何待核验块。
- `实践来源` 只要明确写出 `实践证据不足`，推荐等级只能用 `仅作线索` 或 `已停止/归档`，且 `待核验问题` 不能为 `无`，必须保持完整结构化。
- `当前状态：已停止` 只能配 `已停止/归档`。
- `当前状态：无法确认` 只能配 `仅作线索` 或 `已停止/归档`。

## 模糊未知与唯一豁免

- 以下模式视为模糊未知：`未知`、`未确认`、`未核实`、`未明确`、`未公开`、`未查到`、`不详`、`不明`、`不确定`、`待定`、`待确认`、`待核验`、`无法确认`、`暂无结论`。
- 唯一允许视为已知结论的形式是：`未公开统一数字（以...为准）`。
- 其中 `...` 必须明确指向签约页、合同、后台显示或报价单等正式载体；单写 `未公开` 仍视为未知。
- 资金路径中精确指出缺失清算方只允许保留事实缺口，不适用上述“视为已知”豁免。

## 费用拆分规则

- `平台服务费` 与 `支付通道费` 必须单列。
- `固定费用、提现费、退款费和拒付费` 字段内部必须分别出现且非空：
  - `固定费用：...`
  - `提现费：...`
  - `退款费：...`
  - `拒付费：...`
- 允许明确写 `0`、`无此项`、`未公开统一数字（以签约页为准）`、`未公开统一数字（以合同为准）`、`未公开统一数字（以后台显示为准）`、`未公开统一数字（以报价单为准）`。
- 不接受只写其中一项，不接受泛称 `其他费用`、`其他费`，也不接受在组合费用块中新增第五个冒号子键。
- 普通续行说明可以保留，但不能伪装成新的费用键。

## 官方链接规则

- `官方来源` 至少包含一个可解析的 `http` 或 `https` 链接。
- 生产规则拒绝 `localhost`、`localhost.` 和所有 IP literal。
- hostname 必须是规范化的有效域名，且包含有效的域名点。
- 提取链接时允许剥离尾随中英文标点，例如 `https://example.com。` 仍视为有效链接；`https://。` 仍无效。

## 过期来源处理

- 来源失效、页面下线、截图无法追溯或超过维护周期时，将对应事实标记为待复核。
- 待复核事实可以保留上一版记录，但不能继续作为新的推荐依据。
- 平台停用或产品下线时，推荐等级改为 `已停止/归档`，并保留历史说明。

## 待核验问题格式

当平台档案中的 `待核验问题` 不为 `无` 时，必须使用下面的固定模板：

```markdown
- 缺失事实：还不能确认的具体事实。
- 已查来源：已经检查过的入口、页面、文档、截图和日期。
- 需要询问：下一次需要向谁确认什么问题？
- 可接受证据：什么样的官方页面、文档、合同、报价单、后台截图、申请结果、书面回复、邮件或盖章协议可以完成核验。
```

补充要求：

- 四项都必须非空。
- `缺失事实` 允许写 `无法确认...`，但必须写出具体主题，不能只写 `待确认`、`未知`。
- `需要询问` 必须是长度足够、以 `?` 或 `？` 结尾的具体问题，并包含 `是否`、`可否`、`能否`、`哪些`、`哪类`、`如何`、`多少`、`什么`、`哪一个` 等疑问结构；`请确认是否支持？` 这类泛问不通过。
- `可接受证据` 必须写出具体证据载体；单写 `官方回复` 不通过，`官方书面回复明确写出适用主体、类目和结算路径。` 通过。
- backlog 子字段重复出现视为契约错误。
